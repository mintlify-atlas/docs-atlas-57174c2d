## Overview

Widgets support **bilingual localization** with English and French translations using a simple i18n (internationalization) object pattern. User language preference is stored in localStorage and synchronized across all widgets.

## Language system architecture

The localization system consists of:

1. **i18n object** - Contains all translations organized by language
2. **Translation function** - Helper function to retrieve translated strings
3. **Language toggle** - UI for switching between languages
4. **localStorage** - Persistent language preference
5. **Update function** - Refreshes UI with current language

## Language storage key

All widgets use a consistent localStorage key:

```javascript
const LANG_KEY = "pa_lang";
```

**Possible values:**
- `"en"` - English
- `"fr"` - French (default)

## Translation function

A helper function retrieves translations for the current language:

<CodeGroup>
```javascript Simple strings
// pomodoro/app.js:71-73
function t(key) {
  return i18n[currentLang][key];
}

// Usage:
const pauseText = t("pause");  // "Break" (en) or "Pause" (fr)
const startText = t("btnStart");  // "Start" (both languages)
```

```javascript With parameters
// hydration/app.js:67-70
function t(key, arg) {
  const val = i18n[currentLang][key];
  return typeof val === "function" ? val(arg) : val;
}

// Usage:
const status = t("left", 3);  
// "3 glasses left for your goal" (en)
// "Plus que 3 verres pour l'objectif" (fr)
```
</CodeGroup>

<Note>
  Functions in the i18n object allow for dynamic content like pluralization and parameter interpolation.
</Note>

### How it works

1. **Display current language** - Show "EN" or "FR" in the toggle button
2. **Toggle on click** - Switch between English and French
3. **Persist choice** - Save to localStorage
4. **Update UI** - Call `updateLangUI()` to refresh all translated text
5. **Re-render dynamic content** - Update any computed or dynamic strings

## Cross-widget synchronization

Language changes sync across all open widgets using the `storage` event:

```javascript app.js:90-95
window.addEventListener("storage", (e) => {
  if (e.key === LANG_KEY) {
    currentLang = e.newValue;
    updateLangUI();
  }
});
```

<Tip>
  Change the language in one widget, and all other open widgets update automatically!
</Tip>


## Adding localization to a widget

<Steps>
  <Step title="Create i18n object">
    Define translations for both languages:
    
    ```javascript
    const i18n = {
      fr: {
        title: "Mon Widget",
        btnAction: "Démarrer",
        // ... more French strings
      },
      en: {
        title: "My Widget",
        btnAction: "Start",
        // ... more English strings
      },
    };
    ```
  </Step>
  
  <Step title="Add translation function">
    Create the `t()` helper function:
    
    ```javascript
    function t(key, arg) {
      const val = i18n[currentLang][key];
      return typeof val === "function" ? val(arg) : val;
    }
    ```
  </Step>
  
  <Step title="Load language preference">
    Initialize from localStorage:
    
    ```javascript
    const LANG_KEY = "pa_lang";
    let currentLang = localStorage.getItem(LANG_KEY) || "fr";
    ```
  </Step>
  
  <Step title="Add language toggle">
    Include the toggle in your HTML:
    
    ```html
    <div class="lang-toggle" id="langToggle">EN</div>
    ```
  </Step>
  
  <Step title="Implement toggle logic">
    Add click handler and storage listener:
    
    ```javascript
    function updateLangUI() {
      $("langToggle").textContent = currentLang.toUpperCase();
      $("t-title").textContent = t("title");
      // Update all translatable elements
    }
    
    $("langToggle").addEventListener("click", () => {
      currentLang = currentLang === "fr" ? "en" : "fr";
      localStorage.setItem(LANG_KEY, currentLang);
      updateLangUI();
    });
    
    window.addEventListener("storage", (e) => {
      if (e.key === LANG_KEY) {
        currentLang = e.newValue;
        updateLangUI();
      }
    });
    ```
  </Step>
  
  <Step title="Use translations in code">
    Replace hardcoded strings with `t()` calls:
    
    ```javascript
    // Instead of:
    $("title").textContent = "My Widget";
    
    // Use:
    $("title").textContent = t("title");
    
    // For dynamic content:
    showToast(t("toastSuccess"));
    ```
  </Step>
  
  <Step title="Initialize on load">
    Call `updateLangUI()` when the widget loads:
    
    ```javascript
    updateLangUI();
    ```
  </Step>
</Steps>

## Best practices

<Accordion title="Complete translations">
  Ensure every translation key exists in both languages to avoid undefined errors.
</Accordion>

<Accordion title="Use functions for dynamic content">
  For pluralization, interpolation, or computed strings:
  
  ```javascript
  left: (n) => `${n} item${n !== 1 ? 's' : ''} remaining`
  ```
</Accordion>

<Accordion title="Keep keys semantic">
  Use descriptive key names that indicate purpose:
  
  ```javascript
  // Good
  btnSubmit: "Submit"
  toastSuccess: "✓ Saved successfully!"
  
  // Avoid
  btn1: "Submit"
  msg1: "✓ Saved successfully!"
  ```
</Accordion>

<Accordion title="Update all UI elements">
  Make sure `updateLangUI()` updates:
  - Text content
  - Placeholders
  - ARIA labels
  - Dynamic/computed strings
  - Toast messages
</Accordion>

<Accordion title="Test both languages">
  Verify all features work correctly in both English and French, including:
  - Layout (text might be longer in one language)
  - Pluralization
  - Special characters (é, è, à, etc.)
</Accordion>

<Accordion title="Use emojis consistently">
  Emojis can stay the same across languages:
  
  ```javascript
  fr: {
    toastSuccess: "✅ Objectif atteint !"
  },
  en: {
    toastSuccess: "✅ Goal reached!"
  }
  ```
</Accordion>
