## Overview

The AI Assistant widget provides a full-featured chat interface with support for three major AI providers:

- **Google Gemini** (Free tier available / Pay-as-you-go)
- **Anthropic Claude** (Pay-as-you-go)
- **OpenAI ChatGPT** (Pay-as-you-go)

Features conversation history, system prompts, model selection, streaming responses, and markdown rendering.

<Note>

</Note>

## Setup

<Steps>
  <Step title="Get an API Key">
    Choose one of the following providers and obtain an API key:

    <Tabs>
      <Tab title="Google Gemini">
        **Recommended for beginners** (free tier available)

        1. Go to [Google AI Studio](https://aistudio.google.com/app/apikey)
        2. Sign in with your Google account
        3. Click **"Get API key"** or **"Create API key"**
        4. Copy the key (starts with `AIza...`)

        <Note>
        The free tier includes generous quotas for testing. Pay-as-you-go pricing is very affordable for personal use.
        </Note>
      </Tab>

      <Tab title="Anthropic Claude">
        **Best for advanced reasoning**

        1. Go to [Anthropic Console](https://console.anthropic.com/settings/keys)
        2. Sign in or create an account
        3. Navigate to **API Keys** settings
        4. Click **"Create Key"**
        5. Copy the key (starts with `sk-ant-...`)

        <Warning>
        Anthropic requires adding credits to your account before you can use the API.
        </Warning>
      </Tab>

      <Tab title="OpenAI ChatGPT">
        **Industry standard models**

        1. Go to [OpenAI Platform](https://platform.openai.com/api-keys)
        2. Sign in with your OpenAI account
        3. Click **"Create new secret key"**
        4. Give it a name and copy the key (starts with `sk-...`)

        <Warning>
        Keep your API key secure. Never share it or commit it to version control.
        </Warning>
      </Tab>
    </Tabs>
  </Step>

  <Step title="Add the Widget to iCUE">
    1. Open **iCUE** and navigate to your Xeneon Edge dashboard
    2. Add an **iFrame widget** (size M, L, or XL)
    3. Paste the iframe code from above
    4. The widget will load with a setup prompt
  </Step>

  <Step title="Configure API Key">
    1. Click the **Settings** icon (⚙️) in the top-right corner
    2. Select your AI provider tab (Google, Anthropic, or OpenAI)
    3. Paste your API key in the appropriate field
    4. (Optional) Select a model from the dropdown
    5. Click **Save**
  </Step>

  <Step title="Start Chatting">
    Once configured, the welcome screen will appear with conversation starters. Type your message and press Enter to begin!
  </Step>
</Steps>

If you're using Claude and encounter CORS issues, you can specify a proxy URL:

```text

```

<Warning>

## Features

### Conversation History

The widget automatically saves up to **20 conversations** in local storage:

- View past conversations in the sidebar (toggle with ☰ icon)
- Each conversation shows the first message as the title
- Click any conversation to resume it
- Delete conversations individually with the × button

### API Key Storage

```javascript ai-assistant/app.js
// API keys are stored in sessionStorage for security
function setApiKey(provider, value) {
    const trimmed = (value || "").trim();
    const sessionKey = sessionKeyForProvider(provider);
    const legacyKey = legacyKeyForProvider(provider);
    if (trimmed) safeSet(sessionStorage, sessionKey, trimmed);
    else safeRemove(sessionStorage, sessionKey);
    // Ensure no persistent copy remains.
    safeRemove(localStorage, legacyKey);
}
```

API keys are stored in **sessionStorage** (cleared when browser closes) rather than localStorage for enhanced security. The widget automatically migrates old persistent keys to session storage on first load.

### Streaming Responses

All three providers use Server-Sent Events (SSE) for real-time streaming:

```javascript ai-assistant/app.js
async function streamGemini(contents, systemPrompt, model, key, signal, onChunk) {
    const safeModel = encodeURIComponent(model);
    const url = `${GEMINI_BASE}/${safeModel}:streamGenerateContent?alt=sse`;
    const body = {
        contents,
        generationConfig: { maxOutputTokens: 8192, temperature: 0.7 },
    };
    if (systemPrompt) {
        body.systemInstruction = { parts: [{ text: systemPrompt }] };
    }

    const resp = await fetch(url, {
        method: "POST",
        headers: {
            "Content-Type": "application/json",
            "x-goog-api-key": key,
        },
        body: JSON.stringify(body),
        signal,
        referrerPolicy: "no-referrer",
    });
    // ... SSE parsing logic
}
```

### Markdown Rendering

AI responses are rendered with full markdown support:

- **Code blocks** with syntax highlighting
- **Inline code** formatting
- **Bold**, *italic*, and ***bold+italic***
- Headings (H2-H4)
- Bullet and numbered lists
- Horizontal rules

```javascript ai-assistant/app.js
function parseMarkdown(text) {
    // Escape HTML to prevent XSS
    let s = text
        .replace(/&/g, "&amp;")
        .replace(/</g, "&lt;")
        .replace(/>/g, "&gt;");

    // Fenced code blocks
    s = s.replace(/```(\w*)\n?([\s\S]*?)```/g, (_, lang, code) =>
        `<pre><code${lang ? ` data-lang="${lang}"` : ""}>${code.trimEnd()}</code></pre>`
    );

    // Inline code, bold, italic, etc.
    // ...
}
```

## API Integration Details

### Google Gemini API

**Endpoint:**  
`https://generativelanguage.googleapis.com/v1beta/models/{model}:streamGenerateContent?alt=sse`

**Authentication:**  
API key via `x-goog-api-key` header

**Request Format:**
```json
{
  "contents": [
    { "role": "user", "parts": [{ "text": "Hello" }] },
    { "role": "model", "parts": [{ "text": "Hi there!" }] }
  ],
  "systemInstruction": {
    "parts": [{ "text": "You are a helpful assistant" }]
  },
  "generationConfig": {
    "maxOutputTokens": 8192,
    "temperature": 0.7
  }
}
```

### Anthropic Claude API

**Endpoint:**  
`https://api.anthropic.com/v1/messages`

**Authentication:**  
API key via `x-api-key` header

**Special Header:**
```javascript
headers: {
    "anthropic-dangerous-direct-browser-access": "true"
}
```

This header allows direct browser calls without CORS issues.

**Request Format:**
```json
{
  "model": "claude-sonnet-4-6",
  "max_tokens": 8192,
  "messages": [
    { "role": "user", "content": "Hello" },
    { "role": "assistant", "content": "Hi there!" }
  ],
  "system": "You are a helpful assistant",
  "stream": true
}
```

### OpenAI API

**Endpoint:**  
`https://api.openai.com/v1/chat/completions`

**Authentication:**  
Bearer token via `Authorization` header

**Request Format:**
```json
{
  "model": "gpt-4o",
  "messages": [
    { "role": "system", "content": "You are a helpful assistant" },
    { "role": "user", "content": "Hello" },
    { "role": "assistant", "content": "Hi there!" }
  ],
  "stream": true,
  "max_completion_tokens": 8192
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Error — check your API key">
    **Cause:** Invalid API key or network error

    **Solution:**
    1. Verify your API key is correct (check for extra spaces)
    2. Ensure you have credits/quota remaining with your provider
    3. Check your internet connection
  </Accordion>

  <Accordion title="Claude: Invalid proxy URL">
    **Cause:** Proxy URL doesn't meet HTTPS requirements

    **Solution:**
    - Use HTTPS URLs only
    - Remove any authentication credentials from URL
    - Remove query parameters and hash fragments
    - Or leave proxy URL empty to use direct browser access
  </Accordion>

  <Accordion title="Conversation history disappeared">
    **Cause:** localStorage was cleared or you hit the 20 conversation limit

    **Solution:**
    Conversations are automatically limited to the 20 most recent. Older ones are removed to save space. This is expected behavior.
  </Accordion>

  <Accordion title="Streaming stopped mid-response">
    **Cause:** You clicked the stop button, or network connection was interrupted

    **Solution:**
    This is normal if you stopped generation. Otherwise, check your network connection and retry.
  </Accordion>
</AccordionGroup>
