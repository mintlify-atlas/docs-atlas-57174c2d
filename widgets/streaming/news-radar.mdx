## Overview

News Radar scans RSS feeds for articles matching your keywords. It's designed for tracking specific topics across multiple news sources in real-time.

**Features:**
- Keyword-based filtering of RSS feeds
- Multiple preset sources (Google News, Ars Technica, The Verge)
- Custom RSS feed support
- Article caching for offline viewing
- Click-to-copy article links
- Bilingual (EN/FR) interface

## iframe Code

```html
<iframe src="https://stealthylabshq.github.io/iframe-edge/news-radar/"></iframe>
```

## How It Works

News Radar uses the [rss2json.com](https://rss2json.com) API to fetch and parse RSS feeds, then filters results client-side based on your keyword.

### Architecture

1. **User enters keyword** (e.g., "AI", "climate", "cybersecurity")
2. **Select RSS source** (preset or custom URL)
3. **Click Search** → Fetches feed via rss2json API
4. **Client-side filtering** matches keyword against title + description
5. **Display results** with metadata and copy-to-clipboard

- `pa_newsSource` — Selected source URL or "custom"
- `pa_newsCustomUrl` — Custom feed URL

- `pa_theme` — Theme preference (dark/light)

## Features

### RSS-to-JSON Conversion

The widget uses rss2json.com API to parse feeds:

```javascript news-radar/app.js
const API_BASE = "https://api.rss2json.com/v2/rss_to_json?rss_url=";

async function fetchFeed(keyword) {
    if (!keyword.trim()) { showToast(t("enterKeyword")); return; }

    let rssUrl = sourceSelect.value;
    if (rssUrl === "custom") {
        rssUrl = customUrlInput.value.trim();
        if (!rssUrl) { showToast(t("enterKeyword")); return; }
    }

    $("spinner").classList.add("active");
    $("statusText").textContent = t("loading");
    $("statusCount").textContent = "";

    try {
        const res = await fetch(API_BASE + encodeURIComponent(rssUrl));
        const data = await res.json();

        if (data.status !== "ok" || !data.items) {
            $("statusText").textContent = t("error");
            renderArticles([]);
            return;
        }

        const kw = keyword.toLowerCase();
        const filtered = data.items.filter((item) =>
            (item.title || "").toLowerCase().includes(kw) ||
            stripHtml(item.description || "").toLowerCase().includes(kw)
        );

        localStorage.setItem("pa_newsCache", JSON.stringify(filtered));
        localStorage.setItem("pa_newsKeyword", keyword);

        $("statusText").textContent = filtered.length === 0
            ? t("noResults") + ' "' + keyword + '"'
            : "";

        renderArticles(filtered);
    } catch {
        $("statusText").textContent = t("error");
        renderArticles([]);
    } finally {
        $("spinner").classList.remove("active");
    }
}
```

### Client-Side Filtering

function stripHtml(html) {

### Article Rendering

Each article card displays:
- **Title** (HTML-stripped)
- **Source** (author or feed name)
- **Publication date** (localized)
- **Link button** (copy to clipboard)

```javascript news-radar/app.js
function renderArticles(articles) {
    const list = $("articleList");
    const empty = $("emptyState");

    list.querySelectorAll(".article-card").forEach((c) => c.remove());

    if (!articles || articles.length === 0) {
        empty.style.display = "flex";
        $("statusCount").textContent = "";
        return;
    }

    empty.style.display = "none";
    $("statusCount").textContent = articles.length + " " + t("found");

    articles.forEach((item, i) => {
        const card = document.createElement("div");
        card.className = "article-card";
        setTimeout(() => card.classList.add("pulse"), i * 80);

        const title = document.createElement("div");
        title.className = "article-title";
        title.textContent = stripHtml(item.title || "");

        const meta = document.createElement("div");
        meta.className = "article-meta";

        const source = document.createElement("span");
        source.className = "article-source";
        source.textContent = item.author || item.source || "";

        const date = document.createElement("span");
        date.className = "article-date";
        date.textContent = formatDate(item.pubDate);

        const btn = document.createElement("button");
        btn.className = "btn-link";
        btn.innerHTML = `<svg>...</svg><span>${t("btnLink")}</span>`;

        const link = item.link || "";
        btn.addEventListener("click", () => {
            if (copyToClipboard(link)) {
                showToast(t("linkCopied"));
                btn.classList.add("copied");
                setTimeout(() => btn.classList.remove("copied"), 1200);
            }
        });

        meta.appendChild(source);
        meta.appendChild(date);
        meta.appendChild(btn);
        card.appendChild(title);
        card.appendChild(meta);
        list.appendChild(card);
    });
}
```

Cards appear with a staggered pulse animation (80ms delay per card).

### Clipboard (iCUE-Compatible)

Since `navigator.clipboard` may not work in all iframe contexts, the widget uses a fallback method:

```javascript news-radar/app.js
function copyToClipboard(text) {
    const tmp = document.createElement("textarea");
    tmp.value = text;
    tmp.style.position = "fixed";
    tmp.style.top = "-9999px";
    tmp.style.left = "-9999px";
    tmp.setAttribute("readonly", "");
    document.body.appendChild(tmp);
    tmp.focus();
    tmp.select();
    tmp.setSelectionRange(0, 99999);
    let ok = false;
    try { ok = document.execCommand("copy"); } catch (e) { /* noop */ }
    document.body.removeChild(tmp);
    return ok;
}
```

This creates a temporary textarea, selects the text, and executes the legacy `document.execCommand('copy')` command.

### Result Caching

Results are cached in localStorage for instant restore on reload:

```javascript news-radar/app.js
function restoreState() {
    const savedSource = localStorage.getItem("pa_newsSource");
    if (savedSource) {
        sourceSelect.value = savedSource;
        if (savedSource === "custom") {
            customUrlInput.classList.add("visible");
            customUrlInput.value = localStorage.getItem("pa_newsCustomUrl") || "";
        }
    }

    const savedKw = localStorage.getItem("pa_newsKeyword");
    if (savedKw) $("keywordInput").value = savedKw;

    const cached = localStorage.getItem("pa_newsCache");
    if (cached) {
        try {
            const articles = JSON.parse(cached);
            if (articles.length > 0) {
                renderArticles(articles);
                $("statusText").textContent = t("cached");
            }
        } catch { /* noop */ }
    }
}
```

When the widget loads, it automatically restores:
- Last selected source
- Last keyword
- Last search results (marked "Cache restored")

## API Reference

### rss2json.com API

**Endpoint:**
```
https://api.rss2json.com/v2/rss_to_json?rss_url={encoded_feed_url}
```

**Response Format:**
```json
{
  "status": "ok",
  "feed": {
    "url": "https://example.com/feed",
    "title": "Example Feed",
    "link": "https://example.com",
    "author": "",
    "description": "",
    "image": ""
  },
  "items": [
    {
      "title": "Article Title",
      "pubDate": "2024-01-15 10:30:00",
      "link": "https://example.com/article",
      "author": "Author Name",
      "description": "<p>Article description...</p>",
      "content": "Full content...",
      "enclosure": {},
      "categories": []
    }
  ]
}
```

<Note>
The free tier of rss2json has rate limits. For high-volume use, consider caching or upgrading to a paid plan.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Error loading the feed">
    **Possible causes:**
    - Invalid RSS URL
    - Feed requires authentication
    - rss2json.com API rate limit exceeded
    - Network error

    **Solution:**
    1. Verify the RSS URL is publicly accessible
    2. Test the URL in a browser first
    3. Try a different feed source
    4. Wait a few minutes if rate-limited
  </Accordion>

  <Accordion title="No articles found for keyword">
    **Cause:** Keyword doesn't match any article titles or descriptions in the feed

    **Solution:**
    1. Try a broader keyword (e.g., "tech" instead of "artificial intelligence")
    2. Check spelling
    3. Try a different feed source
  </Accordion>

  <Accordion title="Custom URL doesn't work">
    **Possible causes:**
    - URL is not a valid RSS/Atom feed
    - Feed requires authentication or CORS headers
    - rss2json.com cannot parse the feed format

    **Solution:**
    1. Verify the URL returns XML when opened in a browser
    2. Check for `<rss>` or `<feed>` root elements
    3. Try pasting the URL into rss2json.com directly to test
  </Accordion>

  <Accordion title="Link button doesn't copy">
    **Cause:** Clipboard API blocked in iframe context

    **Solution:**
    The widget uses a fallback clipboard method that works in most environments. If it still fails:
    1. Try clicking the button again
    2. Manually copy the article title and search for it
    3. Check browser console for errors
  </Accordion>
</AccordionGroup>
