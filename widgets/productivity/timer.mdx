The Timer widget is a sophisticated multi-phase countdown timer that lets you create custom timing cycles. Perfect for streaming, presentations, or any workflow requiring timed phases.

## Key Features

- **Multiple phases**: Create unlimited custom timing phases
- **Dual display modes**: Toggle between ring and digital views
- **Editable phases**: Rename phases and adjust durations
- **Color-coded**: Each phase has a unique color for quick recognition
- **Auto-advance**: Automatically moves to next phase when timer ends
- **Cycle tracking**: Loops through all phases continuously
- **Audio alert**: Plays beep sound when phase completes
- **Visual flash**: Screen flash animation on phase end
- **Persistent state**: Saves phases and settings to localStorage
- **Bilingual**: French/English support

## Phase Management

### Adding Phases

```javascript
function addPhase() {
  const colorIdx = st.nextId % PHASE_COLORS.length;
  st.phases.push({
    id: st.nextId++, 
    name: t("newPhaseName"), 
    duration: 5 * 60, 
    colorIdx 
  });
  st.editingIdx = st.phases.length - 1;
  renderPhaseList();
  saveState();
}
```

New phases default to 5 minutes and get the next color in rotation.

### Deleting Phases

Click the trash icon to remove a phase:

```javascript
function deletePhase(idx) {
  if (st.phases.length <= 1) return; // Must have at least 1
  st.phases.splice(idx, 1);
  if (st.currentPhaseIdx >= st.phases.length) {
    st.currentPhaseIdx = st.phases.length - 1;
  }
  renderAll();
  saveState();
}
```

<Note>
You cannot delete the last remaining phase. The widget always requires at least one phase.
</Note>

## Time Input Parsing

The widget accepts multiple time formats:

```javascript
parseTime("25m")      // → 1500 seconds
parseTime("1h30m")    // → 5400 seconds  
parseTime("90")       // → 5400 seconds (interpreted as minutes)
parseTime("1:30:00")  // → 5400 seconds
parseTime("25:00")    // → 1500 seconds
parseTime("1h30m20s") // → 5420 seconds
```

Supported units:

- `h` - hours
- `m` - minutes (not followed by 's')
- `s` - seconds
- `:` - colon-separated (HH:MM:SS or MM:SS)
- Plain number - interpreted as minutes

## Progress Ring Math

```javascript
const CIRC = 2 * Math.PI * 42; // 263.89

function renderTimer() {
  const phase = st.phases[st.currentPhaseIdx];
  const pct = st.timeLeft / phase.duration;
  
  // Ring view
  $("ringProgress").style.strokeDashoffset = CIRC * (1 - pct);
  
  // Digital view  
  $("progressFill").style.width = pct * 100 + "%";
}
```

The SVG circle has:

- `stroke-dasharray="263.89"` - full circle circumference
- `stroke-dashoffset` - animated based on progress
- As offset increases, less of the circle is visible

## Audio Alert

When a phase ends, the widget plays a 3-tone beep:

```javascript
function playEndSound() {
  const ctx = new (window.AudioContext || window.webkitAudioContext)();
  
  function beep(freq, t0, dur) {
    const osc = ctx.createOscillator();
    const gain = ctx.createGain();
    osc.connect(gain);
    gain.connect(ctx.destination);
    osc.frequency.value = freq;
    osc.type = "sine";
    gain.gain.setValueAtTime(0, t0);
    gain.gain.linearRampToValueAtTime(0.22, t0 + 0.02);
    gain.gain.setValueAtTime(0.22, t0 + dur - 0.02);
    gain.gain.linearRampToValueAtTime(0, t0 + dur);
    osc.start(t0);
    osc.stop(t0 + dur + 0.05);
  }
  
  const now = ctx.currentTime;
  beep(660,  now,        0.12);  // E5
  beep(880,  now + 0.18, 0.12);  // A5
  beep(1100, now + 0.36, 0.28);  // C#6
}
```

This creates a pleasant ascending tri-tone alert.

## Phase Transitions

```javascript
function handlePhaseEnd() {
  clearInterval(timerInterval);
  timerInterval = null;
  st.running = false;

  const phaseName = st.phases[st.currentPhaseIdx].name;
  playEndSound();
  triggerEndFlash();

  const nextIdx = (st.currentPhaseIdx + 1) % st.phases.length;
  const isCycle = nextIdx === 0;
  showToast(isCycle ? t("toastCycle") : t("toastEnd", phaseName));

  st.currentPhaseIdx = nextIdx;
  st.timeLeft = st.phases[nextIdx].duration;
  renderAll();
  saveState();
}
```

When a phase ends:

1. Stop the timer
2. Play sound and flash
3. Show toast notification
4. Advance to next phase (or cycle back to first)
5. Reset time to new phase duration
6. Update UI and save state

```javascript
{
  phases: [

  displayMode: "ring" | "digital",
```

`timeLeft` and `running` state are NOT saved. The timer always loads in a stopped state at the beginning of the current phase.

## Advanced Features

### Skip Phase

```javascript
function skipPhase() {
  if (!st.running && st.timeLeft === st.phases[st.currentPhaseIdx].duration) {
    // Not started yet – just advance phase
    handlePhaseEnd();
  } else {
    st.timeLeft = 1;
    handlePhaseEnd();
  }
}
```

Skip works whether timer is running or stopped.

### Format Duration for Display

```javascript
function formatDurationShort(s) {
  const h = Math.floor(s / 3600);
  const m = Math.floor((s % 3600) / 60);
  const sec = s % 60;
  let out = "";
  if (h > 0) out += `${h}h`;
  if (m > 0) out += `${m}m`;
  if (sec > 0) out += `${sec}s`;
  return out || "0m";
}
```

This formats durations as "1h30m" for the edit input pre-fill.
